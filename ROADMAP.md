# Roadmap — Student Engagement Monitor

Система — аналитическая надстройка над LMS (Learning Management System).  
Не собирает данные самостоятельно, а работает с данными активности, экспортированными из LMS.

```
LMS (Moodle, Canvas...) → CSV / Webhook
        ↓
Student Engagement Monitor  ← этот проект
        ↓
Преподаватель принимает решения
```

## 🔜 Планируемые версии

### v0.6.0 — Личный кабинет студента

Студент после входа видит собственные данные, а не пустую страницу.

**Фичи:**
- Персональный дашборд: текущий индекс по каждому курсу с уровнем
- График динамики своей вовлечённости за последние 8 недель
- Список собственных активностей (история)
- Редирект teacher → `/dashboard`, student → `/me`

---

### v0.7.0 — Интерактивный дашборд с фильтрацией

Преподаватель может фильтровать и сортировать студентов прямо на дашборде.

**Фичи:**
- Фильтр по курсу — показать только студентов конкретного курса
- Фильтр по академической группе — студенты группируются по номеру группы (напр. «ИСТ-2025»)
- Фильтр по уровню вовлечённости — показать только critical / low / medium / high
- Сортировка таблицы — по индексу, по имени, по группе
- Поиск по имени / email студента
- URL-параметры — фильтры сохраняются в адресной строке (sharable link)

**Изменения схемы БД:**
- Поле `group` (String, optional) на модели `User` — номер академической группы
- Обновление seed: студентам назначаются группы (напр. «ИСТ-2025», «ИСТ-2024»)

---

### v0.8.0 — Управление курсами + Импорт CSV

Преподаватель может создавать курсы и загружать данные активности из LMS.

**Фичи:**
- Создание курса (название, описание) — форма на дашборде
- Зачисление студентов на курс
- Импорт активности из CSV-файла (формат: `email, activity_type, datetime`)
- Валидация и предпросмотр перед импортом
- После импорта — автоматический пересчёт индексов

**CSV-формат (совместим с типовым экспортом LMS):**
```csv
email,activity_type,course_id,datetime
student1@university.edu,MATERIAL_VIEW,course-is-101,2026-04-01T10:30:00Z
student2@university.edu,ASSIGNMENT_SUBMIT,course-da-201,2026-04-01T14:00:00Z
```

### v1.0.0 — ML-прогнозирование риска

Модель предсказывает вероятность критического падения вовлечённости студента.

**Концепция:**  
На основе паттернов активности за первые 2–3 недели модель предсказывает,
попадёт ли студент в зону критического уровня (`score < 20`) к концу периода.

**Фичи:**
- Risk score (0–100%) для каждого студента на дашборде
- Цветовой индикатор риска: 🟢 низкий / 🟡 средний / 🔴 высокий
- Страница объяснения: какие факторы влияют на риск конкретного студента
- Модель: логистическая регрессия на признаках активности (Python/sklearn → REST API)
  или интеграция с Claude API для интерпретируемого анализа

**Признаки модели:**
- Частота входов за первые 2 недели
- Количество сданных заданий
- Тренд индекса (растёт / падает)
- Процент пропущенных материалов

## Технический долг / улучшения

- Webhook API — внешняя LMS слает события в реальном времени (`POST /api/activity`)
- Email-алерты — уведомление преподавателя при падении индекса ниже порога
- Настройка весов формулы — преподаватель задаёт коэффициенты для своего курса
- Сравнение когорт — сравнение нескольких групп между собой на одном графике
