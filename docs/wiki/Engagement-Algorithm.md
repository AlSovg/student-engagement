# Алгоритм расчёта вовлеченности

`EngagementScore` — числовой показатель от **0 до 100**, отражающий активность студента в рамках курса за недельный период.

---

## Формула

```
Score = Σ (weight_i × normalized_count_i) × 100
```

Где `normalized_count_i = min(count_i / max_i, 1)` — нормализованное количество событий (от 0 до 1).

---

## Веса компонентов

| Компонент             | ActivityType        | Вес  | max (в неделю) |
| --------------------- | ------------------- | ---- | -------------- |
| Частота входов        | `LOGIN`             | 0.10 | 7              |
| Просмотр материалов   | `MATERIAL_VIEW`     | 0.25 | 20             |
| Сдача заданий         | `ASSIGNMENT_SUBMIT` | 0.30 | 5              |
| Участие в обсуждениях | `DISCUSSION_POST`   | 0.20 | 10             |
| Прохождение тестов    | `QUIZ_COMPLETE`     | 0.15 | 5              |

Сумма весов = **1.0**

---

## Пример расчёта

Студент за неделю:

- 5 входов → `min(5/7, 1) × 0.10 = 0.071`
- 12 просмотров → `min(12/20, 1) × 0.25 = 0.150`
- 3 задания → `min(3/5, 1) × 0.30 = 0.180`
- 4 обсуждения → `min(4/10, 1) × 0.20 = 0.080`
- 2 теста → `min(2/5, 1) × 0.15 = 0.060`

**Score = (0.071 + 0.150 + 0.180 + 0.080 + 0.060) × 100 = 54.1**

## Интерпретация

| Диапазон | Уровень               | Цвет         |
| -------- | --------------------- | ------------ |
| 80–100   | Высокая вовлеченность | 🟢 зелёный   |
| 50–79    | Средняя вовлеченность | 🟡 жёлтый    |
| 20–49    | Низкая вовлеченность  | 🟠 оранжевый |
| 0–19     | Критически низкая     | 🔴 красный   |

---

## Реализация

Расчёт выполняется в `src/lib/engagement.ts` (v0.3.0).  
Результаты кешируются в таблице `EngagementScore` с периодом `period` = начало недели (понедельник).  
Пересчёт запускается еженедельно или вручную преподавателем.
